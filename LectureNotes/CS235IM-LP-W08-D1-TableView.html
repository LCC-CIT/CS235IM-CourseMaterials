TableView Applications

Overview

  • There are two ways to use a UITableView in an app:
    • A UITableView can be used with a UIViewController
    • A UITableViewController, which is a UIViewController containing a UITableView can also be used
  • In either approach, the UITableView requires a UITableViewSource to provide the data items that are displayed in the table
  • Tables can optionally have the rows grouped into sections or an index can be added on the side of the table.

Basic Table

  • Look at basic concepts in the Xamarin Guide, Working with Tables and Cells - Table Parts and Functionality
    • UITableViewCell - Each table row is a cell
      • Cells can have different styles, both standard and custom
    • UITableViewSource - Manages the data displayed in the table
  • Walk-throughs: 
    • Code from the Xamarin Guide above GitHub: ios-samples/WorkingWithTables, Part 2 - Populate a Table with Data, BasicTable
      • Behavior:
        • List of plants
        • Clicking on a table cell pops up an Alert with details
      • Code aspects
        • Uses HomeScreen, an instance of a UIViewController, which contains an UITableView instance named table.
        • Uses TableSource, which inherits of UITableViewSource
          • NumberOfSections
          • RowsInSection
          • RowSelected
          • GetCell
            • Uses DequeueReusableCell to recycle UITableViewCell objects. The method will return the next cell with the an identifier string matching the one passed to the method. The underlying structure is FIFO.
              • (A dequeue is a double-ended queue.)
            • If cells haven't been created yet, they are created here.
        • HomeScreen.ViewDidLoad calls CreateTableItems which assigns an instance of TableSource to HomeScreen.table.Source
        • No storyboard
        • Note: Data source doesn't require an array, although it's being converted to an array, List<string> users .ToArray(). Maybe programmer preference? 
    • Look at the Ch. 8 example, SimpleTable - list of Dwarves (no index or detail view)
      • Has a UITableViewController on the storyboard (different from the Xamarin Guide, which has a UIViewController on the storyboard)
      • Has detail text in the row
      • Has an image (star) that changes when the row is clicked

Table with sections and index

  • Look at concepts in the Xamarin Guide, Working with Tables and Cells - Populating a Table with Data
  • Walk-throughs:
    • Code from the Xamarin Guide above, WorkingWithTables - Part 2, Populating a Table with Data
      • Reads list of plants from .txt file, no storyboard
      • Builds sections and an index from the data by using a loop and putting them into a dictionary of lists.
    • My TableView example with an index: TableViewDemo (another list of dwarves)

Cell Styles

  • Default – supports a UIImageView
  • Subtitle – supports a UIImageView and subtitle
  • Value1 – right aligned subtitle, supports a UIImageView
  • Value2 – title is right-aligned and subtitle is left-aligned (but no image)
  • Custom – create a custom layout by subclassing UITableViewCell
